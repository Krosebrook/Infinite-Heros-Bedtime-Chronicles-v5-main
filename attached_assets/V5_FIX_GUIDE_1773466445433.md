# Infinity Heroes V5 — Diagnosis & Fix Guide

**Date:** March 13, 2026  
**App:** `Infinite-Heros-Bedtime-Chronicles-v5` (Replit)  
**Stack:** Expo 54 + React Native + Express + Multi-Provider AI Router  

---

## Architecture Summary (What V5 Actually Is)

V5 is NOT the same app as V1. It's a ground-up rewrite:

| Layer | V1 (Old) | V5 (Current) |
|-------|----------|--------------|
| Frontend | Vite + React (web) | Expo 54 + React Native |
| Backend | Vercel serverless functions | Express.js on Replit |
| AI Text | Gemini only (dead preview model) | Anthropic → Gemini → OpenAI → OpenRouter (fallback chain) |
| AI Images | Gemini only (dead preview model) | Gemini → OpenAI (fallback) |
| TTS | Gemini TTS (raw PCM) | ElevenLabs (9 character voices, MP3) |
| Music | None | Suno (pre-generated tracks per mode) |
| Video | None | OpenAI Sora |
| Storage | IndexedDB (client) | AsyncStorage (client) + PostgreSQL (server) |
| Auth | API key dialog | Replit AI Integrations + Connectors |
| Features | 3 modes, 7 voices | 3 modes, 9 voices, profiles, badges, streaks, parent controls, trophies |

## Step 2: Fix by Symptom

### Symptom: "Stories don't generate"

**Root cause (most likely):** All AI text providers are disconnected.

The story endpoint calls `aiRouter.generateText("story", ...)` which walks a fallback chain: Anthropic → Gemini → OpenAI → Meta-Llama → xAI → Mistral → Cohere. If NONE have both `API_KEY` and `BASE_URL` set, it throws "No AI providers available."

**Fix:**
1. Open Replit → **Settings** → **AI Integrations** (or **Secrets/Environment**)
2. Ensure at least ONE pair is set:

| Minimum for stories | Key | URL |
|---------------------|-----|-----|
| Gemini | `AI_INTEGRATIONS_GEMINI_API_KEY` | `AI_INTEGRATIONS_GEMINI_BASE_URL` |
| OR Anthropic | `AI_INTEGRATIONS_ANTHROPIC_API_KEY` | `AI_INTEGRATIONS_ANTHROPIC_BASE_URL` |

3. The Base URL for Gemini via Replit integrations is typically set automatically by Replit. If it's blank, use: `https://generativelanguage.googleapis.com/v1beta`
4. Restart the server: stop and re-run `npm run server:dev`

**Verify:** `curl -s http://localhost:5000/api/ai-providers` should show at least one provider with `"available": true`.

---

### Symptom: "Images/avatars don't generate" or "always look the same"

**Root cause:** Image generation requires either Gemini (`gemini-2.5-flash-image`) or OpenAI (`gpt-image-1`).

The fallback chain for images is: Gemini → OpenAI.

**"Always look the same" is already fixed in V5** — the art style randomizer (12 styles) is built into `routes.ts` lines 77-94. Each avatar and scene gets a random style. If you're still seeing identical images, it means only one request is succeeding and it's being cached/reused by the client.

**Fix:**
1. Ensure `AI_INTEGRATIONS_GEMINI_API_KEY` + `AI_INTEGRATIONS_GEMINI_BASE_URL` are set (for Gemini image gen)
2. OR ensure `OPENAI_API_KEY` is set (for DALL-E/gpt-image-1 fallback)
3. Check that your Gemini API key has access to the `gemini-2.5-flash-image` model

**Verify:** `curl -s -X POST http://localhost:5000/api/generate-avatar -H "Content-Type: application/json" -d '{"heroName":"Nova","heroTitle":"Test","heroPower":"Light","heroDescription":"A hero"}'` should return `{"image":"data:image/png;base64,..."}`.

### Symptom: "TTS/narration doesn't work"

**Root cause:** ElevenLabs authentication uses Replit Connectors, not a plain env var.

The TTS system (`server/elevenlabs.ts`) calls `https://${REPLIT_CONNECTORS_HOSTNAME}/api/v2/connection` to get the ElevenLabs API key dynamically. If the connector is disconnected, every TTS request fails.

**Fix:**
1. Open Replit → **Settings** → **Connectors**
2. Find **ElevenLabs** and ensure it's connected (green)
3. If it's missing, add it and authorize with your ElevenLabs account
4. Verify `REPLIT_CONNECTORS_HOSTNAME` is set (Replit sets this automatically)

**Alternative fix (bypass Replit Connectors):**
If the connector system is broken, you can hardcode the key. In `server/elevenlabs.ts`, replace the `getCredentials()` function:

```typescript
async function getCredentials() {
  // Direct key fallback if Replit Connectors are broken
  if (process.env.ELEVENLABS_API_KEY) {
    return process.env.ELEVENLABS_API_KEY;
  }
  
  // Original Replit Connectors logic
  const hostname = process.env.REPLIT_CONNECTORS_HOSTNAME;
  // ... rest of existing code ...
}
```

Then set `ELEVENLABS_API_KEY` in your Replit Secrets.

**Verify:** `curl -s -X POST http://localhost:5000/api/tts-preview -H "Content-Type: application/json" -d '{"voice":"moonbeam"}'` should return `{"audioUrl":"/api/tts-audio/xxx.mp3"}`.

### Symptom: "App loads but shows blank/error screen"

**Root cause (likely):** `EXPO_PUBLIC_DOMAIN` not set, or the Expo dev server can't reach the Express backend.

The client resolves the API URL via `getApiUrl()` in `lib/query-client.ts`, which reads `process.env.EXPO_PUBLIC_DOMAIN`. If that's empty, every API call throws "EXPO_PUBLIC_DOMAIN is not set."

**Fix:**
1. Check that `EXPO_PUBLIC_DOMAIN` points to your Replit domain's port 5000
2. This is typically set by the `expo:dev` script: `EXPO_PUBLIC_DOMAIN=$REPLIT_DEV_DOMAIN:5000`
3. If running manually, set it: `export EXPO_PUBLIC_DOMAIN=your-repl-name.replit.dev:5000`

---

### Symptom: "Music doesn't play"

**Root cause:** Missing MP3 files in `assets/music/`.

**Check:**
```bash
ls -la assets/music/
ls -la assets/music/classic/ assets/music/madlibs/ assets/music/sleep/ 2>/dev/null
```

The `getMusicFilePath()` function in `server/suno.ts` looks for MP3s in `assets/music/{mode}/`. If the files got deleted during re-wiring, music silently fails.

---

### Symptom: "Video generation doesn't work"

**Root cause:** Requires `OPENAI_API_KEY` (direct, not integrations) for Sora.

V5 has video generation via OpenAI's Sora model. It's gated behind `OPENAI_API_KEY` and also requires the parent controls `videoEnabled` flag to be true.

**Fix:** Set `OPENAI_API_KEY` in Replit Secrets if you want video. Otherwise, this feature degrades gracefully — the video button won't appear.

## Step 4: Known Code-Level Issues in V5

These are NOT env var problems — they're actual code issues I found during audit.

### Issue A: ElevenLabs Has No Env Var Fallback

`server/elevenlabs.ts` ONLY authenticates via Replit Connectors API. If the connector system breaks (which happens during re-wires), TTS is dead with no fallback. 

**Patch** (add to `server/elevenlabs.ts`, replace lines 3-31):

```typescript
async function getCredentials(): Promise<string> {
  // Priority 1: Direct API key (most reliable)
  if (process.env.ELEVENLABS_API_KEY) {
    return process.env.ELEVENLABS_API_KEY;
  }

  // Priority 2: Replit Connectors
  const hostname = process.env.REPLIT_CONNECTORS_HOSTNAME;
  const xReplitToken = process.env.REPL_IDENTITY
    ? 'repl ' + process.env.REPL_IDENTITY
    : process.env.WEB_REPL_RENEWAL
    ? 'depl ' + process.env.WEB_REPL_RENEWAL
    : null;

  if (!hostname || !xReplitToken) {
    throw new Error('ElevenLabs not configured: set ELEVENLABS_API_KEY or connect via Replit Connectors');
  }

  const connectionSettings = await fetch(
    'https://' + hostname + '/api/v2/connection?include_secrets=true&connector_names=elevenlabs',
    {
      headers: {
        'Accept': 'application/json',
        'X_REPLIT_TOKEN': xReplitToken
      }
    }
  ).then(res => res.json()).then(data => data.items?.[0]);

  if (!connectionSettings?.settings?.api_key) {
    throw new Error('ElevenLabs connector not available');
  }
  return connectionSettings.settings.api_key;
}
```

### Issue B: Client Story Response Handling Assumes Server Returns Correct Shape

In `app/story.tsx` line 566, the client does `setStoryData(data as StoryFull)` without validation. If any AI provider returns malformed JSON that passes the server's basic checks but has missing fields, the UI crashes silently.

**Low priority** — the server already validates `parts` array existence. But if you see blank story screens after successful generation, this is the cause.

### Issue C: Scene Image Persistence Was Just Fixed

The dead code triage from today (March 13) shows `saveStoryScene` was restored. Verify this is in the deployed code — if your Replit is running an older commit, scene images won't persist between sessions.

## What "Re-Wiring" Likely Did

Based on the git history and code state, the re-wiring was probably a Replit Agent session that:

1. **Disconnected AI Integrations** — Replit's AI connectors get reset when the project config changes
2. **May have changed the ElevenLabs connector** — if the agent touched `.replit` or `replit.nix`
3. **Merged settings systems** (documented in DEAD-CODE-TRIAGE.md) — this was intentional and correct
4. **Wired up audio routes** (documented) — also intentional
5. **May have changed the server start command** — check `.replit` file's `run` command

The V5 code itself is solid. The multi-provider AI router, the art style randomizer, the response schema enforcement, the child safety prompts — all present and correct. The problem is almost certainly at the infrastructure/connector layer, not the code layer.
